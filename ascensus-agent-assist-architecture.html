The Three-Layer Stack

Each layer has a distinct role. Failures map cleanly to layers — that's why this picture matters.

Agent Assist — What Associates See

The associate-facing tool. Real-time suggestions, knowledge retrieval, confidence scores, next actions.

What it does
The UI an Ascensus associate works in during a participant call. Surfaces suggested responses, retrieved policy snippets, confidence badges, and recommended next actions. Associate stays in control — they review, edit, and approve every outbound response.
Key capabilities
  • Real-time suggested response panel
  • Knowledge snippet retrieval with source citation
  • Confidence scores and amber/red warning badges
  • Next-action recommendations (e.g. "open beneficiary form in Salesforce CRM")
  • Inline edit + send
Failure domain mapping
Agent Assist itself rarely fails — it's a presentation layer. When something looks broken here, the root cause is almost always in ARK (retrieval, intent, tool calls) or downstream in Salesforce CRM / Chorus.

ARK — McKinsey Agentic Runtime

Deploys and manages AI agents, models, tool integrations, and memory. Built on Kubernetes. Powers the agents inside Agent Assist.

What it does
The runtime that hosts agents, routes intents, calls LLMs, executes tool integrations, and manages memory. Everything Agent Assist "thinks" runs here. Kubernetes-based — agents scale and deploy independently.
Key capabilities
  • Agent lifecycle management (deploy, version, roll back)
  • Intent classification and routing
  • Knowledge retrieval over policy corpora
  • Tool calls into Salesforce CRM, Chorus, and internal APIs
  • Memory and session state
  • LLM gateway
Failure domain mapping
Most failures live here: RETRIEVAL_AMBIGUOUS, INTENT_LOW_CONFIDENCE, TOOL_CALL_FAILED, FALLBACK_TRIGGERED. Diagnosed via Langfuse traces, cross-referenced with New Relic for infra signals.
│
▼

SS&C Blue Prism Chorus — Operational Backbone

BPM platform. Manages business workflows, compliance tracking, case management, document handling. Executes the operational follow-through after AI recommends.

What it does
The system of record for case execution. After Agent Assist recommends and the associate approves, Chorus drives the workflow — assigning steps, tracking SLAs, capturing compliance artifacts, routing documents. The "did the work actually happen?" layer.
Key capabilities
  • Business process orchestration
  • Case management and SLA tracking
  • Compliance audit trail
  • Document handling and storage
  • Workflow assignment and escalation
Failure domain mapping
Chorus failures are operational, not AI — missing case state, stuck workflows, compliance gaps. Triaged in the Chorus operations console; rarely require Langfuse.

Where Things Break — The Three Failure Domains

Every issue an associate-engineer triages maps to one of these. Knowing which one collapses the diagnostic time.

⚠️ RETRIEVAL_AMBIGUOUS
Domain
Knowledge Retrieval (ARK layer)
What it looks like
Confidence drops below 80%. Two conflicting documents retrieved. Associate sees amber badge.
Example
RMD eligibility — missing employment status and ownership %.
How to find it
Langfuse trace — retrieval step, similarity score, chunk count.
Fix
Add missing context docs, tune similarity threshold, improve query rewriting.
🔴 TOOL_CALL_FAILED
Domain
Salesforce CRM Tool Call (ARK → Salesforce)
What it looks like
FALLBACK_TRIGGERED. Associate sees safe fallback message. Debug panel shows timeout.
Example
Beneficiary update — Salesforce write call times out at 3002ms.
How to find it
Langfuse trace — tool call step, latency, error payload. Cross-check New Relic.
Fix
Structured Azure DevOps ticket. Tag as Known Issue Pattern if 3+ occurrences.
⚠️ INTENT_LOW_CONFIDENCE
Domain
Intent Classification (ARK layer)
What it looks like
Classifier confidence below threshold. System unsure which intent to route to.
Example
Ambiguous question that could be hardship withdrawal or standard distribution.
How to find it
Langfuse trace — intent step, raw confidence score.
Fix
Add training examples for this intent, adjust classification threshold.

Key Talking Points — Quick Reference

Compressed cheatsheet for fast recall.

01Your Core Narrative

Ascensus Agent Assist is a three-layer system. ARK — McKinsey's agentic runtime on Kubernetes — hosts the agents that think. Agent Assist is the associate-facing surface that speaks. Chorus, SS&C Blue Prism's BPM platform, executes the operational follow-through. My job sits at the seam between them: when an associate flags a bad answer, I pull the Langfuse trace, walk the pipeline step by step — intent, retrieval, tool call, generation — identify which failure domain owns the issue, and document it in Azure DevOps. Recurring patterns get tagged as Known Issue Patterns and escalated. The discipline is precise: don't guess the layer, read the trace.

03Anticipated Questions

What's the difference between ARK and Agent Assist?
ARK is the runtime — Kubernetes-hosted agents, models, tool integrations, memory. Agent Assist is the UI an associate works in. ARK thinks, Agent Assist speaks. When Agent Assist looks broken, the cause is almost always in ARK or downstream.
Walk me through how you'd diagnose a bad suggested response.
Pull the Langfuse trace for that turn. Walk the steps in order: intent classification (was confidence high?), retrieval (which docs, what similarity scores?), tool calls (did anything time out or 5xx?), LLM generation (was the prompt sane?). The first step that looks off is the failure domain. Cross-reference New Relic if I suspect an infra issue.
What's a Known Issue Pattern and when do you tag one?
When the same failure shape recurs 3+ times — same intent, same retrieval gap, same tool timeout — I tag the Azure DevOps work item as a Known Issue Pattern. That promotes it from one-off triage to something the engineering team prioritizes structurally.
Where does Chorus come in?
Chorus is the operational backbone. After the associate approves a recommendation, Chorus runs the actual business workflow — case management, SLA tracking, compliance artifacts, document routing. AI failures live in ARK; operational failures (stuck case, missed SLA) live in Chorus.
How do you decide whether to escalate?
Two triggers: recurrence (3+ instances of the same pattern → Known Issue Pattern, escalate to the owning team) or severity (a single TOOL_CALL_FAILED that blocks a high-value flow like beneficiary updates → escalate immediately with the Langfuse trace and New Relic correlation attached).

04The Stack in One Sentence Each

  • Agent Assist — the associate-facing UI that surfaces AI suggestions, knowledge snippets, and next actions in real time.
  • ARK — McKinsey's Kubernetes-based agentic runtime that hosts agents, intents, retrieval, tool calls, and memory.
  • Chorus — SS&C Blue Prism's BPM platform that drives case workflows, SLAs, and compliance after AI recommends.
  • Langfuse — AI observability; the source of truth for tracing what happened inside any given turn.
  • New Relic — infrastructure monitoring; correlates latency, errors, and resource health across services.
  • Azure DevOps — where issues get documented, tagged, and prioritized — including Known Issue Patterns.
  • Cursor — the IDE used for code-level investigation, fix prototyping, and config edits.
  • Salesforce CRM — the integration point ARK reaches into for participant records and case state.
